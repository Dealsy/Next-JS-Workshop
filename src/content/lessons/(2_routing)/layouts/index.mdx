Next.js 13+ introduced a powerful new way to handle layouts using the App Router. Layouts allow you to share UI between multiple pages while maintaining state and avoiding re-renders.

## What are Layouts?

A layout is a UI that is shared between multiple pages. You can define layouts using the app directory's special `layout.tsx` file.
Layouts preserve state, remain interactive, and do not re-render when the page changes.

### Root layout:

A root layout is the layout that sits at the top of your app directory, this layout is applied to all pages
created within the app.

<ThemeImage
  darkSrc="/images/dark_rootlayout.svg"
  lightSrc="/images/light_rootlayout.svg"
  alt="Layout diagram showing how layouts work in Next.js"
  size="xs"
/>

The root layout is where you will typically define your Navigation in the form of a header and or footer,
you may also define other things such as:

- Theme
- Authentication
- Analytics
- Sidebar

```tsx
// app/layout.tsx
export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <body>
        <nav>{/* Your navigation */}</nav>
        <main>{children}</main>
        <footer>{/* Your footer */}</footer>
      </body>
    </html>
  );
}
```

The children prop is where all pages that are created within your app will be rendered.

### 1. Nested Layouts

Layouts can be nested within the app, this allows you to create a specific layout for a specific page.

You can create nested layouts by adding a `layout.tsx` file in child folders:

<ThemeImage
  darkSrc="/images/dark_layout.svg"
  lightSrc="/images/light_layout.svg"
  alt="Layout diagram showing how layouts work in Next.js"
  size="medium"
/>

When you create a nested layout, you need to create it in the route folder that you want it to apply to,
this nested layout will have a slightly different syntax to the root layout.

Here is what it looks like:

```tsx
// app/blog/layout.tsx
export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <div>
      <aside>{/* Blog-specific sidebar */}</aside>
      {children}
    </div>
  );
}
```

<Alert type="info">
  <span>
    It's important to note that the nested layout will not override the root
    layout, but rather it sits within it.
  </span>
  <span>
    If you were to add a header or sidebar to the nested layout, it would still
    have the root layout header and sidebar.
  </span>
</Alert>

### 2. State Preservation

- Layouts maintain state when navigating between pages
- Components inside layouts stay mounted and preserve their state
- No unnecessary re-renders when switching pages

### 3. Partial Rendering

- Only the page components update during navigation
- The layout remains static.

## Best Practices

1. **Root Layout Requirements**

   - Must contain `html` and `body` tags
   - Required in the app directory
   - Applies to all routes

2. **Data Fetching**
   - Layouts can fetch their data
   - Data fetching occurs in parallel for improved performance
   - Whilst we can fetch data in the layouts, its still always reccomeneded to fetch data closest to the component that needs it.

```tsx
// app/layout.tsx
async function getUser() {
  const res = await fetch("...");
  return res.json();
}

export default async function Layout({
  children,
}: {
  children: React.ReactNode;
}) {
  const user = await getUser();
  return (
    <div>
      <header>
        <p>Welcome {user.name}</p>
      </header>
      {children}
    </div>
  );
}
```

## Common Patterns

### 1. Persistent Navigation

```tsx
export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <div className="flex min-h-screen">
      <nav className="bg-gray-100">{/* Navigation items */}</nav>
      <main className="flex-1">{children}</main>
    </div>
  );
}
```

### 2. Authentication

You should place your authentication logic in the root layout, this will ensure that the user is authenticated for all pages within the app.

```tsx
// app/layout.tsx
export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <div>
      <AuthProvider>{children}</AuthProvider>
    </div>
  );
}
```

### 3. Themes

Themes can be applied to the root layout, this will ensure that the theme is applied to all pages within the app.

```tsx
// app/layout.tsx
export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <div>
      <ThemeProvider>{children}</ThemeProvider>
    </div>
  );
}
```
