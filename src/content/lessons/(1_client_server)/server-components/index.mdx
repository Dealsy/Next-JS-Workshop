Server Components are a fundamental concept in Next.js that allows you to render components on the server by default. This approach brings several benefits including improved performance, reduced client-side JavaScript, better SEO and a better Developer Experience (DX).

## What are Server Components?

Server Components are React components that execute and render entirely on the server.

Server Components must be `async` functions as of Next 15.

They allow developers to:

- Access server-side resources directly
- Keep sensitive data and logic on the server
- Support streaming and progressive rendering
- Render HTML on the server
- Allow for optional caching
- Revalidate routes from the server
- Data fetching

All pages are Server Components by default in Next.js

```tsx
export default function ServerComponent() {
  return (
    <div>
      <h1>Hello from the Server!</h1>
    </div>
  );
}
```

## Key Benefits

### 1. Zero Client-Side JavaScript

Server Components don't ship any JavaScript to the client by default. This means:

- Faster page loads
- Lower bandwidth usage
- Better performance on low-end devices

### 2. Direct Backend Access

You can directly access backend resources:

```tsx
// Access database directly in a Server Component ( The client doesn't have access to the database )
export default async function UserProfile({ userId }: { userId: string }) {
  const user = await db.users.findUnique({
    where: { id: userId },
  });

  return <div>Hello, {user.name}!</div>;
}
```

## 3. Automatic Code Splitting

Server Components help with automatic code splitting by keeping server-only code separate from client code.

## 4. Data Fetching

Server Components can fetch data from the server and pass it to the client, the amount of syntax required is
significantly less than in Client Components.

For example, to fetch data client side, you would need to do something like this:

```tsx
"use client";

const useData = () => {
  const [data, setData] = useState<any>(null);
  // Handle loading state
  const [isLoading, setIsLoading] = useState(true);
  // Handle error state
  const [error, setError] = useState<Error | null>(null);

  useEffect(() => {
    const fetchData = async () => {
      try {
        const response = await fetch("/api/data");
        const result = await response.json();
        setData(result);
      } catch (err) {
        setError(err instanceof Error ? err : new Error("An error occurred"));
      }
      setIsLoading(false);
    };

    fetchData();
  }, []);

  return { data, isLoading, error };
};
```

Using it in the client component

```tsx
"use client";
export default function DataComponent() {
  const { data, isLoading, error } = useData();

  if (isLoading) return <div>Loading...</div>;
  if (error) return <div>Error: {error.message}</div>;

  return <div>{data.name}</div>;
}
```

We have to account for loading and errors and make sure we get the useEffect
hook to fetch the data correctly without accidentally causing race conditions or forgetting
any other logic.

On the server we can change this to simply be:

```tsx
export default async function Page() {
  // Fetching data from a fetch function
  const response = await getData(); // [!code highlight]

  if (!response.ok) {
    throw new Error(`HTTP error! status: ${response.status}`);
  }

  const data = await response.json();

  return <div>{data.name}</div>;
}
```

Since we are fetching on the server, the data will be fetched before the client has to fully render,
this means we can avioid useEffect hell and just simply call the endpoint directly.

## When to use Server Components

Use Server Components when:

- You need to access backend resources directly
- The component doesn't need interactivity
- You want to keep sensitive data/logic on the server
- You want to reduce client-side JavaScript

```tsx
// Good use case for a Server Component
export default async function ProductList() {
  const products = await fetchProducts(); // Direct database access

  return (
    <div>
      {products.map((product) => (
        <div key={product.id}>
          <h2>{product.name}</h2>
          <p>{product.description}</p>
        </div>
      ))}
    </div>
  );
}
```

<Alert type="info">
  If your component needs interactivity, you need to use a Client Component.

Client components allow you to use browser APIs and React hooks, these are not
available in Server Components.

</Alert>

## Best Practices

1. **Start with Server Components**

   - Use Server Components by default
   - Only switch to Client Components when needed for interactivity

2. **Proper Component Boundaries**

   - Keep interactive parts isolated in Client Components
   - Don't make entire pages Client Components

3. **Data Fetching**
   - Fetch data in Server Components when possible
   - Use React Suspense for loading states

```tsx
// Good pattern: Server Component with isolated client interactivity
export default async function Page() {
  const data = await fetchData();

  return (
    <div>
      <ServerRenderedContent data={data} />
      <ClientInteractiveWidget /> {/* Only this part is interactive */}
    </div>
  );
}
```

## Common Pitfalls

1. **Mixing Server and Client Code**

```tsx
// ✅ Correct: Use browser API in Client Component
"use client";
export function WindowSize() {
  const [width, setWidth] = useState(window.innerWidth);
  // ... rest of the component
}

// ❌ Wrong: Using browser APIs in Server Component
export default async function ServerComponent() {
  const width = window.innerWidth; // This will throw an error!
  return <div>Width: {width}</div>;
}
```

2. **Over-using Client Components**

```tsx
// ❌ Wrong: Making entire page a Client Component
"use client";
export default function Page() {
  return (
    <div>
      <StaticHeader /> {/* Could be a Server Component */}
      <InteractiveForm /> {/* Needs to be Client */}
      <StaticFooter /> {/* Could be a Server Component */}
    </div>
  );
}

// ✅ Correct: Only make interactive parts Client Components
export default async function Page() {
  return (
    <div>
      <StaticHeader />
      <ClientForm /> {/* Only this needs 'use client' */}
      <StaticFooter />
    </div>
  );
}
```

3. **Server components must be async functions**
