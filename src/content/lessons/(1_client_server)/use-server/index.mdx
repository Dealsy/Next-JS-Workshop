The `'use server'` directive in Next.js enables Server Actions, allowing you to run code exclusively on the server, even when it's called from client components.

Server actions should be used to perform an action, such as creating a todo item, updating a user's profile, or deleting a comment.
You should not use server actions to perform a query, such as fetching data from a database.

<Alert type="warning">
You should think of server actions as a public API that you can call from your client components.

Next JS still leaves authentication and authorization checks up to you.
So it is important that you implement these checks in your server actions if
you want to secure your server actions.

</Alert>

## When to Use Server Actions

Use the `'use server'` directive when you need to:

- Mutate data in your database
- Perform server-only operations
- Handle form submissions securely
- Handle any server-side CRUD operations

## Example

In this example, the server action is defined in a separate file and imported into the component.
this is the typical way that you would use server actions.

```tsx
// action.ts
"use server";

import { db } from "@/lib/db";

export async function createTodo(title: string) {
  const user = await getCurrentUser();

  if (!user.id) {
    throw new Error("Unauthorized");
  }

  // This code only runs on the server
  await db.todo.create({
    data: { title },
  });
}

// In a separate file using the server action

import { createTodo } from "./action";

export default function TodoForm() {
  return (
    <form action={createTodo}>
      <input type="text" name="title" />
      <button type="submit">Add Todo</button>
    </form>
  );
}
```

You can also use `'use server'` in a component to open a door to the server.

Here is an example:

```tsx
export default function TodoList() {
  async function deleteTodo(formData: FormData) {
    "use server";

    const user = await getCurrentUser();

    if (!user.id) {
      throw new Error("Unauthorized");
    }

    // This code only runs on the server
    const id = Number(formData.get("id"));
    todos = todos.filter((t) => t.id !== id);
    revalidatePath("/");
  }

  return (
    <form action={deleteTodo}>
      <button type="submit">Delete Todo</button>
    </form>
  );
}
```

## Best Practices

1. Keep Server Actions Focused

   - One action should do one thing
   - Return minimal necessary data

2. Error Handling

   - Return meaningful error messages
   - Validate input data server-side ( You can use Zod to validate input data server-side)

3. Security Considerations
   - Never expose sensitive data or credentials
   - Always validate and sanitize inputs
   - Use proper authentication checks

## Common Gotchas

- Server Actions must be async functions
- Can't mix client and server code in the same file
- Server Actions can't access client-side APIs
- Server Actions are API calls and will cause a network request
