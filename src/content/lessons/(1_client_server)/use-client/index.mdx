The `'use client'` directive is used to tell Next JS that this component is a client component, and that it should
render on the client side.

Client Components are a fundamental concept in Next.js that allow you to make parts of your application interactive. They're essential when you need to use browser APIs, handle user interactions, or manage client-side state.

Becuase server components are the default in Next JS, you need to specify when you need a client component, which you can do
by adding the `'use client'` directive to the top of the file.

## When to Use Client Components

Use the `'use client'` directive when your component needs to:

- Use browser APIs (localStorage, window, etc.)
- Add interactivity and event handlers
- Use React hooks (useState, useEffect, etc.)
- Handle form submissions
- Add client-side routing logic

## Example

```tsx
"use client";

import { useState } from "react";

export default function Counter() {
  const [count, setCount] = useState(0);

  return <button onClick={() => setCount(count + 1)}>Count: {count}</button>;
}
```

## Best Practices

1. Keep client components as small as possible

   - Only mark components as client when they need interactivity
   - Split client and server logic into separate components

2. Push state management down the tree

   - Keep client components at the leaves of your component tree
   - Parent components can remain server components

3. Use Server Components by default
   - Start with server components
   - Convert to client components only when needed
   - This optimizes your application's performance

## Common Gotchas

- You can't use `'use client'` in the middle of a file
- Client components can't import server components (but server components can import client components)
